hexo建立博客，git上传到远成仓库

hexo
  • git
  • hexo

自从入了hexo这个坑之后，享受了诸多甜头，也踩了无数的坑，几天吧踩过的坑来总结一下。

话说今天的hexo又踩了一个坑，本来想把自己写好的项目直接deploy到hexo中，但是，不知为何js代码总是出错，于是乎，我直接在git上更改了代码，结果就碰到了地雷，在hexo d代码的时候总是d不上，问了度娘好久，还是没有找到原因。心一横，备份一下配置文件，Ctrl+a，再shift+delete+enter。OK，这个故事才刚刚开始。

hexo的安装

1. npm install -g hexo-cli

然后开始初始化hexo

2. hexo init  
3. npm install

做完这两步就初始化好了，然后的步骤要小心,首先生成静态页面。

4. hexo g

然后创建远程主机仓库，就是在你的github上建立一个新仓库仓库名字的规则是：

仓库的名称 = 你注册的昵称 + github.io

然后重要的一步来了，一定要执行以下这个代码，否则hexo d 有时会报错

npm install hexo-deployer-git --save

如何配置_config.yml文件这里就不多说了。然后就往github上deploy就行了

hexo d

OK，再把自己的配置文件弄回来，终于好了。

git上传到远成仓库

首先关联你的远成仓库：

git remote add origin git@server-name:path/repo-name.git

如果远程仓库有内容，就先pull一下,这样能够保证远成仓库里的东西，本地仓库都有，并且千万注意，远成仓库需要有README.md

git pull

然后添加你本地仓库里的文件。如果太多就选择一下都添加：

git add -A

然后提交

git commit -m '文件修改的描述'

然后就可以push了,第一次push一般使用

git push -u origin master

之后的平时就可以使用

git push origin master

展示自己的作品

自己的作品push到github上之后，就可以用github的HTML Preview对你写的页面进行view，下面是地址，经常找这个地址

http://htmlpreview.github.io/?

然后你view结果得地址，就是你展示作品的地址了。可以写在文章的Markdown链接里面，就可以连接到你的地址了。
